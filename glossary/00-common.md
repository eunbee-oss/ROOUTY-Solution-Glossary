# 공통 용어

ROOUTY TMS의 **모든 메뉴에서 공통으로 사용되는 용어**입니다. 루티를 처음 접한다면 이 문서의 "핵심 도메인 개념"부터 읽어보세요.

## 핵심 도메인 개념

루티 업무 흐름은 **주문 등록 → 배차 계획 → 주행(모니터링) → 보고**로 이어집니다. 아래 개념이 전 과정에서 반복 사용됩니다.

> 📦 **제품 라인업 참고** — 이 용어집은 현재 운영 주력인 **Roouty v1 (TMS, tms.roouty.io)** 기준입니다. 별도 제품군으로 재구축 중인 **루티프로 v2**, 콜드체인 온도관제 **콜드아이**, 경로 최적화 **엔진**이 있습니다. (출처: Roouty Handbook)

| 용어 | English | 정의 |
|---|---|---|
| 배차 | Dispatch | 주문을 차량(기사)에 배정하는 행위 |
| 자동 배차 | Auto Dispatch | 최적화 알고리즘으로 주문을 차량에 자동 배정하는 방식 |
| 자가 배송 | Self Delivery | 배차 시스템을 거치지 않고 화주가 직접 배송을 수행하는 방식 |
| 주행 | Route / Trip | 배차 확정 후 차량이 경로를 따라 운행하는 것 |
| 경로ID | Route ID | 배차(주행) 건에 시스템이 부여하는 고유 식별자 (예: 20260206R962231) |
| 회전 | Trip / Round | 차량이 출발지에서 출발해 배송 후 복귀하는 1회 운행 단위 |
| 다회전 | Multi Round | 한 차량이 출발지 기준으로 여러 회전을 도는 방식 💬 다회전 배차에서는 일부 기능을 사용할 수 없습니다 (Beta) |
| 밀크런 | Milk Run | 한 차량이 여러 지점을 순회하며 수거·배송을 함께 수행하는 운송 방식 |
| 권역 | Zone | 배차 시 차량에게 담당시킬 지역 단위 |
| 용적량 | Capacity | 주문·차량의 적재량 지표. 무게/부피/수량 등 최대 3개 기준(용적량1~3)으로 관리 |
| 특수 조건 | Special Condition | 주문과 차량을 매칭하는 조건 태그 (예: 냉장, 1.4톤). 조건이 걸린 주문은 같은 조건의 차량에만 배차됨 |
| 화주사 | Shipper | 배송 처리를 의뢰한 제품의 소유자·의뢰 업체 |
| 중개사 | Broker | 주문을 담당하는 영업사원·대리인 등 중개 주체 |
| 납품처 | Delivery Point | 반복 배송하는 고정 배송지 (설정에서 사전 등록) |
| 하위 구분 | Sub-category | 납품처를 추가로 분류하기 위한 태그 💬 배차 계획 수립 시 참고 정보로 활용됩니다 |
| PoD | Proof of Delivery | 작업 완료 증빙 (인수증·작업 완료 사진 등) |
| 관제 | Control | '모니터링' 메뉴의 사내 통용 명칭 — 배송 중 현황(위치·상태)을 실시간으로 지켜보는 것 |
| 라스트마일 배송 | Last-mile Delivery | 고객에게 도달하는 최종 구간의 배송. 루티(TMS)가 관리하는 영역 |
| 자동 주문 / 수동 주문 | Auto / Manual Order | 최적화 대상 주문(자동)과 기사 직접 지정 주문(수동). 주문관리에서 탭·URL(`/manage/order/auto·manual`)로 구분 |
| 임시주문 | Temporary Order | 엑셀 대량 업로드 시 등록 확정 전 상태의 주문 — 행 검증·수정 후 확정하는 2단계 처리 |
| 마스터주문 | Master Order | 정기/반복 배송을 위한 주문 템플릿 |
| 지오펜싱 | Geofencing | 기사 앱이 배송지 근접을 자동 감지해 도착/완료 처리를 트리거하는 기능 |
| 백오피스 | Backoffice | 내부 운영/CS용 관리 도구 — 고객사 관리·데이터 조회·이슈 대응 |

## 전역 내비게이션 (GNB)

화면 최상단에 항상 표시되는 메뉴입니다.

| 구분 | 용어 | English | 정의 |
|---|---|---|---|
| 메뉴 | 주문관리 | Order Management | 주문의 등록, 조회, 수정, 취소 등 주문 전반을 관리 |
| 메뉴 | 배차 계획 | Dispatch Planning | 배차 옵션을 설정하고 주문과 차량을 선택하여 배차를 계획 |
| 메뉴 | 모니터링 | Monitoring | 배차 목록 조회 및 주행 중인 배차 주문의 실시간 모니터링 |
| 메뉴 | 보고서 | Report | 전체·차량별 주행 기록 관리 및 현황 확인 |
| 메뉴 | 정산 | Settlement | 운송 비용·정산 관리 (현재 비활성 · 준비 중) |
| 메뉴 | 메시지 관리 | Message Management | 고객·기사·거래처 문자 메시지 발송·예약·자동화 |
| 아이콘 | 설정 ⚙️ | Settings | 조직·리소스·기준 정보·PoD 관리 (톱니바퀴 아이콘) |
| 아이콘 | 공지사항 📢 | Notice | 서비스 공지 페이지 |
| 아이콘 | 도움말 ❓ | FAQ | 자주 묻는 질문 페이지 |
| 표시 | 사용방법 문의 | Support Contact | 고객 지원 연락처 (1533-0441) |

## 조회 · 표 공통 기능

모든 목록 화면에서 동일하게 동작하는 기능입니다.

| 구분 | 용어 | English | 정의 |
|---|---|---|---|
| 버튼 | 조회하기 | Search | 설정한 조건으로 목록을 검색 |
| 버튼 | 초기화 | Reset | 검색 조건을 기본값으로 되돌림 |
| 버튼 | 다운로드 | Download | 조회된 목록을 파일로 내려받기 |
| 필터 | 조회 기간 | Search Period | 조회 기준 날짜 범위 (오늘/1주일/1개월/3개월 또는 직접 지정) |
| 표시 | 조회 항목 | Search Results Count | 검색 결과로 조회된 건수 |
| 옵션 | N개씩 보기 | Rows per Page | 한 페이지에 표시할 행 수 (10/20/30/40/50/300/500) |
| 아이콘 | 표 항목 조회 설정 ⚙️ | Table Column Settings | 목록에 표시할 컬럼을 노출/미노출로 관리 (표 우측 상단 톱니바퀴) |
| 필드 | 노출 목록 | Visible Columns | 테이블에 표시되는 컬럼 목록 |
| 필드 | 미노출 목록 | Hidden Columns | 테이블에 표시되지 않는 컬럼 목록 |
| 필드 | 소속 팀 | Team | 데이터(차량·주문·매니저 등)가 속한 팀 |

## 화면 용어 ↔ 내부 코드 매핑 (CS·개발 소통용)

CS와 개발이 같은 대상을 다른 이름으로 부르는 경우를 대비한 매핑표입니다. (출처: Roouty Handbook · 코드 표기는 개발 문서 기준)

| 화면 용어 | 내부 코드/영문 | 비고 |
|---|---|---|
| 배차 계획 | `route-plan` | 핸드북·코드 표기 |
| 권역 | `area` | 권역 옵션 파라미터: `distAreaType` |
| └ 권역 무관 | `none` | 기본값 |
| └ 기본 권역 | `dedicated` | |
| └ 유연한 권역 | `flexible` | |
| 배차 기준 · 차량 수 최소 | `dispatchType=minVehicle` | |
| 배차 기준 · 업무시간 최소 | `dispatchType=minWorktime` | |
| 균등 배차 · 균등 업무 시간 | `equalizeBy=worktime` | |
| 균등 배차 · 균등 주문 개수 | `equalizeBy=workload` | |
| 유류비 · 통행료 자동 계산 | `isToll` | 경로 계산 시 통행료 반영 여부 |
| 다회전 | `multi round` | 다회전 배차는 `/v2/route/multi/*` 로 분기 |
| 주행대기 | `activated` | 모니터링(관제) 상태 코드 |
| 주행중 | `processing` | |
| 주행종료 | `completed` | |
| 저장된 배차 | `temporary` | |
| 보류 | `skip` | 주문 보류 처리 |
| 임시주문 | `temporary order` | 엑셀 업로드 확정 전 주문 |
